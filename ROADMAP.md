# Roadmap

## Software validation

- Expand benchmark tasks beyond hand-written demonstrations.
- Add adapters for established symbolic-regression benchmark formats.
- Add wall-clock, memory, and proposal-efficiency reporting.
- Add independent installation reports from external users.

## Recommended public development before journal submission

- Publish the repository and keep it public for more than six months.
- Use issues, pull requests, tagged releases, and public discussions.
- Document at least one real research workflow using the software.
- Seek independent reproduction by at least one external researcher.

## Hardware validation - explicitly future work

- Implement an isolated Stratum-compatible test server.
- Connect a physical SHA-256 miner on a private network.
- Validate byte order and accepted-share handling.
- Measure energy, latency, share rate, rejected shares, and candidate throughput.
- Compare against CPU-generated seeds under equal evaluation budgets.
- Publish raw power-meter and network traces.
