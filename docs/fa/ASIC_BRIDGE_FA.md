# پل ماینر و محدودیت سخت‌افزار

## آنچه این نسخه انجام می‌دهد

این نسخه می‌تواند:

- هدر ۸۰ بایتی تولید کند؛
- هش دوبل را روی پردازنده مرجع محاسبه کند؛
- هدف را با ترتیب بایتی صحیح بررسی کند؛
- سهم خارجی شامل پیشوند هدر، نانس، هش و هدف را اعتبارسنجی کند؛
- سهم پذیرفته‌شده را به بذر مستقل تغییر تبدیل کند.

نمونه بررسی سهم:

```bash
hashwave verify-share '{"header_hex":"...","nonce":1,"digest_hex":"...","target_hex":"..."}'
```

## آنچه هنوز انجام نشده است

این نسخه شامل سرور کامل پروتکل استراتوم نیست. برای اتصال مستقیم دستگاه‌هایی مانند انت‌ماینر باید این اجزا اضافه و روی سخت‌افزار واقعی آزمایش شوند:

1. مدیریت اتصال و احراز هویت کارگر؛
2. تخصیص عدد اضافی برای جلوگیری از تکرار هدر؛
3. ساخت پیام اعلام کار؛
4. دریافت و بازسازی سهم ارسالی؛
5. تنظیم سختی متغیر؛
6. مدیریت پایان فضای ۳۲ بیتی نانس؛
7. ثبت تأخیر، سهم تکراری و سهم قدیمی؛
8. اتصال سهم تأییدشده به والد و نسل صحیح.

## نکته ترتیب بایتی

بایت‌های خام خروجی هش دوبل با نمایشی که معمولاً در مرورگرهای بلاک دیده می‌شود معکوس هستند. نسخه‌های قبلی شرط صفر را روی سمت اشتباه بررسی می‌کردند. نسخه ۱٫۰ عدد هش را با ترتیب کوچک‌انتها می‌سازد و مستقیماً با هدف مقایسه می‌کند.

## نرخ سهم

اگر توان دستگاه `R` هش بر ثانیه و شرط پذیرش `q` بیت باشد:

```text
shares_per_second = R / 2^q
```

اما این نرخ فقط نرخ بذرهای قابل تحویل را بیان می‌کند. برای اثبات برتری اقتصادی باید این معیار اندازه‌گیری شود:

```text
پاسخ معتبر به‌ازای هر کیلووات‌ساعت
```

این اندازه‌گیری بدون ماینر، وات‌متر و اتصال واقعی قابل انجام نیست.
