# معماری ریاضی موج هش

## ۱. هدر کار و اثبات هش

هدر کار ۸۰ بایت است:

```text
B = version || previous || state_root || timestamp || bits || nonce
```

هش کار:

```text
D = SHA256(SHA256(B))
```

برای سازگاری عددی با روش بیت‌کوین، بایت‌های خام هش به‌صورت عدد صحیح ۲۵۶ بیتی با ترتیب کوچک‌انتها تفسیر می‌شوند:

```text
h = LE256(D)
```

سهم معتبر است اگر:

```text
h <= T
```

که در آن `T` هدف سهم است.

در آزمایش‌های محلی می‌توان هدف مصنوعی با `q` بیت صفر تعریف کرد:

```text
T(q) = 2^(256-q) - 1
```

احتمال تقریبی پذیرش هر هش:

```text
P(accept) = 2^(-q)
```

## ۲. جداسازی بذر تغییر

هش پذیرفته‌شده مستقیماً به دستور تغییر تبدیل نمی‌شود. بذر جدید با جداسازی دامنه ساخته می‌شود:

```text
u = SHA256(
    domain
    || D
    || nonce
    || parent_hash
    || state_root
    || generation
)
```

این کار دو هدف دارد:

- تصمیم‌های جست‌وجو از قالب و محدودیت‌های سهم جدا می‌شوند؛
- یک سهم برای یک والد یا نسل دیگر قابل بازپخش نیست.

## ۳. نمایش حالت

هر برنامه یک درخت عبارت است. نمونه:

```text
("add", ("mul", ("c", 3), ("x",)), ("c", 1))
```

شناسه حالت:

```text
s(p) = SHA256("HashWave/expr/v1" || serialize(canonical(p)))
```

فرم کانونی، عبارت‌های ساده و تکراری را کاهش می‌دهد؛ مانند:

```text
x + 0  ->  x
x - x  ->  0
x XOR x  ->  0
1 * x  ->  x
```

## ۴. انتقال حالت

فرزند از والد با تغییر محدود ساخته می‌شود:

```text
p(t+1) = Mutation(p(t), u)
```

تغییرها شامل این موارد‌اند:

- تغییر یک ثابت؛
- افزودن یک عملگر؛
- حذف یک زیرعبارت؛
- تعویض ترتیب شاخه‌ها؛
- جایگزینی عملگر؛
- افزودن عبارت کوچک؛
- تقاطع زیرعبارت دو والد.

محدودیت اندازه و عمق:

```text
nodes(p) <= Nmax

depth(p) <= Dmax
```

## ۵. ارزیابی

برای برنامه `p` و نقاط آموزش:

```text
Etrain(p) = (1/n) * Σ |p(x_i) - y_i|
```

برای اعتبارسنجی:

```text
Eval(p) = (1/m) * Σ |p(v_i) - z_i|
```

تابع انتخاب:

```text
J(p) = Etrain(p)
     + λv * Eval(p)
     + λc * nodes(p)
     - λn * novelty(p)
     - λs * support(p)
```

در نسخه نهایی:

```text
λs = 0
```

زیرا پشتیبانی مسیر در آزمایش‌های قبلی مزیت آماری پایداری نشان نداد.

## ۶. سلول معنایی

خروجی برنامه روی آموزش و اعتبارسنجی به‌صورت بردار رفتار ثبت می‌شود:

```text
V(p) = [p(x1), ..., p(xn), p(v1), ..., p(vm)]
```

سلول معنایی:

```text
C(p) = Trunc96(SHA256("HashWave/semantic/v1" || V(p)))
```

در هر سلول فقط بهترین برنامه نگهداری می‌شود. این کار از اشغال پرتو جست‌وجو توسط برنامه‌های هم‌رفتار جلوگیری می‌کند.

## ۷. حافظه ارزیابی

هش حالت کلید کش است:

```text
Cache[s(p)] = evaluation(p)
```

در نتیجه یک عبارت تکراری دوباره اجرا نمی‌شود. آمار نهایی تعداد پیشنهادها، ارزیابی‌های منحصربه‌فرد و برخوردهای کش را جدا گزارش می‌کند.

## ۸. جداسازی عملگرها بر اساس مسئله

هر مسئله فهرست عملگرهای مجاز دارد. برای مثال:

```text
چندجمله‌ای:
add, sub, mul, neg

قدر مطلق:
add, sub, mul, neg, abs

مسئله بیتی:
add, sub, xor, and, or, neg
```

این محدودیت از پیدا شدن پاسخ‌های تصادفی و شکننده با عملگرهای نامرتبط جلوگیری می‌کند.

## ۹. حافظه پُربعد فارسی

هر متن پس از یکسان‌سازی حروف فارسی و حذف نشانه‌های اضافی، به ویژگی‌های زیر تبدیل می‌شود:

- واژه؛
- دوتایی واژه‌ها؛
- دوحرفی، سه‌حرفی و چهارحرفی؛
- مترادف‌های محدود و از پیش تعریف‌شده.

هر ویژگی به یک بردار دودویی ۸۱۹۲ بیتی نگاشت و بردارها با رأی اکثریت ترکیب می‌شوند. شباهت با فاصله همینگ محاسبه می‌شود:

```text
sim(a,b) = 1 - Hamming(a,b) / dimension
```
